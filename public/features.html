Change templates

In order to change the style of a page without the need to delete it a website editor should be able to change the template of a node

@cms Change template

  1. Given a page exists using the "Page A" template
  2. And I am editing the content
  3. When I follow "Change Template"
  4. And I choose "Page C"
  5. And I press "Change"
  6. And I visit the content page
  7. Then the page should be in the "Page C" template

Cannot change template for "Home" template

  1. Given a page exists using the "Home" template
  2. And I am editing the content
  3. Then I should not see "Change Template"

Cannot change template as parent only allows one sub template

  1. Given a page exists using the "Page B" template
  2. And that page has a "Page C" parent
  3. And I am editing the content
  4. Then I should not see "Change Template"

Prevent Template Change if sub content not allowed in new template

  1. Given a page exists using the "Page A" template
  2. And that page has "Page A" subpages
  3. And that page has a "Page A" parent
  4. And I am editing the content
  5. When I follow "Change Template"
  6. And I choose "Page C"
  7. And I press "Change"
  8. Then I should see "Template cannot be changed as sub content is not allowed in this template"
  9. When I visit the content page
  10. Then the page should be in the "Page A" template

Groups access control

To control which users will be able to modify certain parts of the website a website administrator will be able to set the access level that groups have to branches of the content tree

Set Content Permission

  1. Given I am signed in as a website administrator
  2. When I am editing content
  3. Then I should be able to set the permissions on that content

Permissions

  1. Given content's <Permission> is set to "staff" and "students"
  2. Then only users in the "staff" and "students" should be able to <Actions> content
  3. And users not in the "staff" and "students" should not be able to <Actions> content

Inherited Permissions

  1. Given content exists with permissions set
  2. When a child of that content is created
  3. Then by default the child should have the same permissions as it's parent

Administrators

  2. Then I should be able to carry out all actions regardless of group permissions

Manage assets

In order to allow easy access to documents whilst editing a website editor will be able to easily upload, browse and modify documents

Upload an Asset

  1. Given I am using the asset library
  2. When I upload a file
  3. And enter tags
  4. Then it should appear in the asset library
  5. When I follow "Show"
  6. Then I should see "Viewing"
  7. When I follow "Edit"
  8. Then I should see "Editing Asset"

Browse Assets

  1. Given files have been uploaded to the asset library
  2. When I am using the asset library
  3. Then I should be able to browse assets by content type
  4. And I should be able to browse assets by tags

@javascript Insert an asset

  1. Given files have been uploaded to the asset library
  2. When I am editing content
  3. And I click the editor "Insert Asset" button
  4. And follow "Images"
  5. And I click "Add" on an Asset
  6. Then the "Image" asset should appear in the content editor

Manage Content Tree

In order to provide up to date information in a structured manner on the website a website editor will be able to manage the content in a tree structure

Background

  1. Given the website has been populated with content based on the site map

Browse Content Tree

  1. Given I am on the content admin page
  2. Then I should see a list of the roots
  3. When I click on a root
  4. Then I should see a list the of the root’s children
  5. When I click on a child
  6. Then I should see a list of the child’s children

Create Root Content

  1. Given I am on the content admin page
  2. Then I should be able to create a new root
  3. And I should see the root listed within the roots

Create Sub Content

  1. Given I am on the content admin page
  2. When I click on a root
  3. Then I should be able to create a new child
  4. And I should see the child listed within the root’s children

Delete Content

  1. Given I am on the content admin page
  2. Then I should be able to delete content
  3. And the content and all of it’s sub content will be removed from the website

Reorder Children

  1. Given I am on the content admin page
  2. When I click on a root
  3. Then I should be able change the order of the root’s children

@javascript Move Child to another Parent

  1. Given I am on the content admin page
  2. When I click on a root
  3. Then I should be able to move a child content to another parent
  4. And I should see the child listed within the other parent’s children

Publish Content

In order to control when content is available on the website a website administrator will be able to define the time period for which content is published

Publish content

  1. Given content exists
  2. When I publish the content
  3. Then the content should be visible on the website
  4. When I save content as draft
  5. Then the content should not be visible on the website

Publish content for set period

  1. Given content exists
  2. And I publish content between "20th July 2010" and "30th July 2010"
  3. And today is "15th July 2010"
  4. Then the content should not be visible on the website
  5. When today is "25th July 2010"
  6. Then the content should be visible on the website
  7. When today is "5th August 2010"
  8. Then the content should not be visible on the website

Publish content for set period

  1. Given today is "5th August 2010"
  2. And published content exists with publish to date: "30th July 2010"
  3. Then the content should not be visible on the website
  4. When I am editing the content
  5. And I clear the publish to date
  6. Then the content should be visible on the website
